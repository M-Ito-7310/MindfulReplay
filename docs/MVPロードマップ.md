### プロジェクト概要

本プロジェクトは、YouTubeの偶然的な再視聴を、能動的な学びと行動の必然的なサイクルに変えるモバイルアプリケーションを開発することを目的とします。ユーザーが動画から得た学びを記録し、それを具体的な行動に結びつけるための仕組みを提供します。

### MVP（最小限の機能）開発の要件定義

MVPフェーズでは、**「動画の保存」「タイムスタンプ付きメモ」「シンプルリマインダー」「検索」**という、3つの主要ペルソナ（ケンタ、ユミ、マサオ）の共通課題を解決するコア機能に絞り込みます。

#### 1. ユーザー認証機能
* **目的**: ユーザーが個人のデータを安全に管理するための基盤を構築します。
* **要件**:
    * メールアドレスとパスワードによる新規アカウント登録、およびログイン機能。
    * パスワードはハッシュ化して安全に保存します。

#### 2. 動画保存機能
* **目的**: ユーザーが関心を持ったYouTube動画を、アプリケーション内に収集・管理できるようにします。マサオさんの「情報の散逸」という課題を解決します。
* **要件**:
    * YouTubeのURLを貼り付けると、バックエンドがYouTube Data API v3を介して動画タイトルやサムネイルを自動取得し、データベースに保存する。
    * 取得データ: `youtube_id`, `title`, `thumbnail_url`。

#### 3. メモ機能（タイムスタンプ付き）
* **目的**: 動画視聴中に得た学びを、ケンタさんの「多忙な日常」でも効率的に記録できるようにします。
* **要件**:
    * 動画の特定箇所（タイムスタンプ）に紐づけてメモを記録できる機能。
    * メモはテキスト形式で、`content`と`timestamp_sec`を格納します。
    * UI上では、タイムスタンプをタップすることでYouTube動画の該当箇所にジャンプできる仕組みを実装します。

#### 4. タグ付け・検索機能
* **目的**: ケンタさんの「情報の整理不足」やユミさんの「知識の断片化」を解決し、マサオさんの「アナログな管理」から脱却させます。
* **要件**:
    * メモに対して複数のタグを自由に設定できる。
    * メモ内容、タグ、動画タイトルを対象とした全文検索機能。

#### 5. シンプルリマインダー機能
* **目的**: 記録した学びを、設定したタイミングで再提示することで、「偶然」を「必然」に変えるコア機能を実現します。
* **要件**:
    * メモと通知日時を紐づけてリマインダーを設定できる。
    * 設定日時になると、プッシュ通知サービス（例: Firebase Cloud Messaging）を通じてユーザーに通知を送信する。

### 技術スタック

モバイルアプリケーションとして迅速に開発し、将来的なAI機能の拡張を考慮した技術スタックを選定します。

* **フロントエンド（モバイル）**: **React Native**
    * **理由**: 単一のコードベースでiOS/Android両方に対応できるため、開発コストを最小限に抑えられます。開発者コミュニティが成熟しており、UIコンポーネントやライブラリが豊富です。ネイティブに近いパフォーマンスと、プッシュ通知などのモバイル固有機能へのアクセスが可能です。
* **バックエンド**: **Node.js (Express)**
    * **理由**: 非同期I/Oに強く、APIサーバーとして高いパフォーマンスを発揮します。フロントエンドと同じJavaScript/TypeScriptで開発できるため、チーム全体の学習コストが低く、生産性が向上します。
* **データベース**: **PostgreSQL**
    * **理由**: 堅牢で、将来的にユーザー数やデータ量が増加しても安定した運用が可能です。リレーショナルデータベースとしてスキーマが明確なため、データの一貫性を保ちやすく、ビジネスロジックの複雑化にも耐えられます。
* **インフラ**: **Google Cloud Platform (GCP)**
    * **理由**: YouTube Data APIとの親和性が高く、APIキーの管理や、将来的なAI機能（Vertex AIなど）の導入が容易です。Firebase Cloud Messaging（FCM）を利用したプッシュ通知もシームレスに実装できます。

### ロードマップ

MVP開発から将来の機能拡張まで、3つのフェーズで段階的に進めます。

#### フェーズ1: MVP開発（3ヶ月）
* **目的**: コア機能の実装と市場投入。
* **タスク**:
    * 基本の認証、動画保存、メモ、タグ付け、検索、シンプルリマインダー機能のバックエンドAPI開発。
    * React NativeによるiOS/Android向けモバイルアプリのUI/UX開発。
    * YouTube Data APIおよびFirebase Cloud Messagingとの連携。
* **成果物**: ユーザーが動画を保存し、メモ・タグを付けて、リマインダーを設定できる最小限のアプリケーション。

#### フェーズ2: 機能拡張（MVPローンチ後6ヶ月）
* **目的**: ユーザーからのフィードバックに基づき、ペルソナごとの特定ニーズに応える。
* **タスク**:
    * **ユミ向け**: テーマ管理機能の実装。複数の動画とメモをテーマごとに統合するバックエンドAPIと、それに対応するフロントエンドUIを開発します。
    * **ケンタ向け**: タスク管理機能の実装。メモをタスクとして登録し、進捗を管理できる機能を追加します。
    * UI/UXの改善: マサオさんのニーズに応えるため、ユーザーテストを通じて、より直感的でシンプルな操作性を追求します。

#### フェーズ3: AI連携と高度化（MVPローンチ後12ヶ月）
* **目的**: アプリケーションの差別化を図り、より深い学習体験を提供する。
* **タスク**:
    * **AIによる動画要約機能**: YouTube動画の文字起こしデータや説明文をAIが分析し、要点を自動で生成する機能を実装します。これはケンタさんの「時短」ニーズを解決する重要な機能です。
    * **学習曲線リマインダー**: エビングハウスの忘却曲線をモデルに、メモした内容をユーザーが忘れかけたタイミングで自動的にリマインドする機能を実装します。
    * **パーソナライズされたコンテンツ推薦**: ユーザーの行動データや学習テーマを分析し、最適な動画や学習パスをAIが提案する機能を追加します。

### 今後の展望（AIエンジニアへの期待）

フェーズ3以降は、AI技術がこのアプリケーションの成否を握る鍵となります。特に以下の領域で、AIエンジニアの専門知識と創造性が不可欠です。

* **自然言語処理（NLP）**: YouTube動画の文字起こしデータから、文脈を理解した上で正確な要約やキーワードを抽出するモデルを構築してください。これは、ケンタさんの「時短・効率化」のニーズに直接応える核心的な機能となります。
* **データ分析と機械学習**: ユーザーの学習行動パターンを分析し、忘却曲線に基づいたリマインダーアルゴリズムを開発してください。また、ユミさんの「学習の道筋」や「進捗の可視化」を実現するために、パーソナライズされたコンテンツ推薦エンジンを構築してください。

AI技術を活用することで、このアプリケーションは単なるメモツールを超え、ユーザーの学びと成長を強力にサポートする、唯一無二のプラットフォームへと進化できると確信しています。
